# MCP セキュリティ一次審査

候補MCPサーバーを以下のrubricで評価し、**構造化レポート**を出力する。
これは **一次審査（自動）** であり承認ではない。最終承認は人間（セキュリティ担当）が行い、有効化は Owner が `allowedMcpServers` に登録した時点で初めて成立する（PII組織のため）。

## 入力として集めるもの
- MCPサーバー名 / 提供元（URL・リポジトリ）/ 用途 / 触れるデータ分類

## 評価rubric（各項目を 低 / 中 / 高リスク で判定し根拠を書く）
1. **提供元の信頼性** … 公式 / 著名コミュニティ / 個人製・無名
2. **通信形態** … ローカル stdio / リモートHTTP（＝データが外部に出る。高リスク寄り）
3. **要求スコープ・アクセス範囲** … 何にアクセスするか、過剰要求でないか
4. **認証・トークンの扱い** … secret の保存・送信方法
5. **prompt injection 面** … 外部の未検証コンテンツを取り込むか
6. **データ分類適合（最重要）** … 顧客PIIに触れ得るか / 外部SaaSへ送るか
7. **供給網** … バージョン固定・更新管理・メンテ状況

## 出力フォーマット
- 各rubric項目のリスク判定と根拠
- 総合判定（承認推奨 / 条件付き / 非推奨）と付帯条件
- 確認できなかった点（人間レビューで要確認の宿題）

## 原則
- 不明・確認不能な点は「高リスク寄り」に倒す。
- このスキルの判定だけで承認しない（人間承認＋Owner登録が必須）。
