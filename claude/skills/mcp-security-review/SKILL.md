# MCP セキュリティ一次審査（rubricベース・スコアリング付き）

候補MCPサーバーを、実際に起きたインシデント由来の**9項目チェックリスト**で審査し、点数化してレポートする。
**外部サービス・アカウント・APIトークンは不要**（Claude が候補MCPの公開ソース／マニフェスト／ツール説明を読んで評価する。既知CVEは公開DBで照合する）。
**これは一次審査（自動）であり承認ではない。** 最終承認は人間（セキュリティ担当）が行い、有効化は Owner が managed の `allowedMcpServers` に登録した時点で成立する。

## 進め方
1. 候補MCPの情報を集める：提供元（リポジトリURL）・マニフェスト/設定・ツール定義（名前と説明）・依存パッケージ。
   - 公開リポジトリなら WebFetch で README／ソース／`package.json` 等を読む。社内製ならローカルを読む。
   - **顧客PII・本番秘密情報は一切渡さない**（審査対象は“MCPの定義”であって自社データではない）。
2. 既知の脆弱性を照合：`vulnerablemcp.info` や CVE で、その MCP／依存に未修正の既知問題が無いか確認する。
3. 下の9項目を 0/1/2 点で採点し、クリティカルゲートを判定する。
4. 総点・判定・申し送りをレポートする。

## 検査項目（9つ・実インシデント由来）
各項目を **2=低リスク / 1=中 / 0=高リスク** で採点し、根拠（該当箇所）を必ず添える。

| # | 項目 | 確認ポイント | 由来インシデント |
|---|---|---|---|
| A | ツール説明の汚染 | ツール名／説明／引数説明に隠し指示・難読化・「他ツールを無視」等が無いか | Tool Poisoning (Invariant Labs, 2025-04) |
| B | 可変なツール定義 | 承認後に定義/configが変わり得るか。バージョン/SHA固定・自動更新の有無 | MCPoison (CVE-2025-54136) |
| C | config改変経路 | 外部コンテンツ取込 → `mcp.json` 等を書き換える設計でないか | CurXecute (CVE-2025-54135) |
| D | 越権・データ持ち出し | アクセス範囲は最小か／取得データの外部送信（隠しBCC・外部API）／同居MCPのデータを読めるか | GitHub MCP heist, Postmark |
| E | コマンド実行 / RCE | STDIOでシェル実行・入力サニタイズ・**既知CVE照合** | mcp-remote (CVE-2025-6514), aws-mcp (CVE-2025-5277) 等 |
| F | SSRF / 不審な取得 | 内部/プライベートIP・クラウドメタデータEPへ到達できるか、URL検証 | server-puppeteer |
| G | 認証情報・トークン | 保存／送信／ログ出力の扱い、要求権限が過剰でないか | Nx トークン窃取 |
| H | 供給元・供給網 | 提供元（公式/個人）・メンテ体制・実績・依存の健全性・単独メンテのインサイダーリスク | Nx, Postmark |
| I | 通信形態・PII適合（最重要） | ローカルstdio／リモートHTTP（外部送信）、顧客PIIに触れ得るか／外部SaaSへ送るか | 自社方針・横断 |

## スコアリング
- 合計 ＝ 各項目の点の総和（**満点 18**）。
- **★クリティカルゲート**（1つでも該当＝総点に関係なく「非推奨」）：
  - E に未修正の既知 RCE / CVE
  - I で顧客PIIが外部に出る
  - D に隠し送信／明確な exfil 経路
  - A に明確な注入指示

| 総点 | クリティカルゲート | 判定 |
|---|---|---|
| 15–18 | 非該当 | ✅ 承認推奨 |
| 10–14 | 非該当 | ⚠️ 条件付き（要対策・追加確認） |
| 0–9 | — | ❌ 非推奨 |
| — | 該当 | ❌ 非推奨（即） |

## 出力フォーマット
- 候補MCPの概要（名前・提供元・通信形態）
- 9項目それぞれ：点（0/1/2）＋根拠
- クリティカルゲート判定（該当の有無と内容）
- **総点 / 判定**
- 人間レビューへの申し送り（確認できなかった点・付帯条件）

## 原則
- 不明・確認不能な点は **0点（高リスク寄り）** に倒す。
- このスキルの判定だけで承認しない（**人間承認＋Owner登録が必須**）。
- 外部APIやアカウントは使わない（自社方針：外部依存を増やさない）。
